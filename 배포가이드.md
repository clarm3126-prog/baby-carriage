# 베베카 배포 가이드 — 30분 완성 🚀

> 백엔드 없이 Vercel 서버리스로 CoolSMS + 토스페이먼츠를 연동하는 방법입니다.

---

## 📁 파일 구조

```
bebeca-api/              ← Vercel에 배포할 폴더
  api/
    otp.js              ← SMS 인증번호 발송/검증
    payment/
      confirm.js        ← 토스 결제 최종 승인
    health.js           ← 서버 상태 확인
  package.json
  vercel.json

bebeca-mvp-v3.html      ← 고객용 모바일 웹 (이 파일만 서빙하면 됨)
```

---

## STEP 1 — 계정 3개 만들기 (모두 무료)

### 1-1. Vercel 계정
→ https://vercel.com/signup  
GitHub 계정으로 가입하면 편합니다.

### 1-2. CoolSMS 계정
→ https://coolsms.co.kr  
가입 후 **[개발자] → [API Key 관리] → 키 생성**  
- API Key, API Secret 복사 보관  
- **[설정] → [발신번호 관리]** → 본인 휴대폰 번호 등록 (심사 1~2일)  
- 충전: 최소 5,000원 (SMS 약 250건)

### 1-3. 토스페이먼츠 계정
→ https://developers.tosspayments.com  
가입 후 **[테스트 연동]** 탭  
- **클라이언트 키** (test_ck_...) 복사  
- **시크릿 키** (test_sk_...) 복사  
- 실서비스 전환 시 [라이브 연동] 탭에서 사업자 등록증 제출

---

## STEP 2 — Vercel 배포 (5분)

### 방법 A: Vercel CLI (터미널)
```bash
# 1. Node.js 설치 확인
node --version   # v18 이상 권장

# 2. Vercel CLI 설치
npm install -g vercel

# 3. bebeca-api 폴더로 이동
cd bebeca-api

# 4. 의존성 설치
npm install

# 5. 배포 (처음엔 로그인 필요)
vercel

# → 질문에 모두 Enter (기본값)
# → 배포 완료 후 URL 확인 (예: https://bebeca-api-xxx.vercel.app)
```

### 방법 B: Vercel 웹 대시보드 (코딩 없음)
1. vercel.com → **New Project**
2. **bebeca-api 폴더**를 드래그앤드롭 or GitHub 연결
3. Deploy 클릭

## STEP 3 — 환경변수 설정 (Vercel 대시보드)

Vercel 프로젝트 → **Settings → Environment Variables**

| 변수명 | 값 | 예시 |
|--------|-----|------|
| `COOLSMS_API_KEY` | CoolSMS API Key | `NCS…` |
| `COOLSMS_API_SECRET` | CoolSMS API Secret | `DQNX…` |
| `COOLSMS_SENDER` | 등록된 발신번호 (하이픈 없이) | `01012345678` |
| `TOSS_SECRET_KEY` | 토스 시크릿 키 | `test_sk_…` |

저장 후 **Redeploy** 클릭 (환경변수는 재배포 후 적용)

## STEP 4 — 프론트엔드 설정

`bebeca-mvp-v3.html` 파일 상단 CONFIG 수정:

```javascript
const CONFIG = {
  KAKAO_JS_KEY: 'abc123...',                        // 카카오 JS 키
  API_BASE: 'https://bebeca-api-xxx.vercel.app',    // Vercel 배포 URL
  TOSS_CLIENT_KEY: 'test_ck_D4yKeq5b...',           // 토스 클라이언트 키
};
```

## STEP 6 — 카카오 로그인 설정

1. https://developers.kakao.com → 내 애플리케이션 → 앱 추가
2. **플랫폼 → Web → 사이트 도메인** 등록
   - 개발: `http://localhost` 또는 파일 열면 `null`
   - 배포: `https://your-domain.com`
3. **카카오 로그인 활성화** ON
4. **동의항목**: 닉네임(필수), 프로필 사진(선택), 이메일(선택) 체크
5. **JavaScript 키** 복사 → CONFIG에 입력

## 테스트 시나리오

### SMS 인증 테스트
- API_BASE 비워두면 → 인증번호 `1234` 고정 (실제 문자 발송 없음)
- API_BASE 입력하면 → 실제 문자 발송

### 토스 결제 테스트 (test_ck_ 키 사용 시)
| 항목 | 테스트 값 |
|------|---------|
| 카드번호 | 아무 숫자 16자리 (예: 4242 4242 4242 4242) |
| 유효기간 | 아무 미래 날짜 (예: 12/26) |
| CVC | 아무 3자리 (예: 123) |
| 비밀번호 | 아무 2자리 (예: 00) |

→ 실제 결제 없이 성공/실패 시나리오 테스트 가능

## 문제 해결

**Q. CoolSMS 발신번호 등록이 안 돼요**  
→ 개인 번호는 본인 명의 휴대폰만 등록 가능. 심사 1~2 영업일 소요.

**Q. 토스 결제창이 안 열려요**  
→ `file://` 로컬 파일에서는 토스 SDK가 동작하지 않을 수 있어요.  
→ VS Code의 Live Server 또는 `npx serve .` 로 localhost로 열어야 합니다.

**Q. CORS 오류가 나요**  
→ vercel.json의 headers 설정이 적용됐는지 확인. 재배포 필요.

**Q. 카카오 로그인이 안 돼요**  
→ 플랫폼에 현재 접속 도메인이 등록됐는지 확인.  
→ 로컬 테스트: `http://localhost` 등록 필요.
